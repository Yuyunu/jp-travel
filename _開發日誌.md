# 開發日誌 — jp-travel

## 2026-05-04 — v0.1.0 首版（3 個場景）

### 範圍
- 規格：N5 程度旅遊 PWA、3 tab（對話 / 單字 / 測驗）、TTS、深淺色、和風配色
- 場景：先做 3 個（機場 / 餐廳 / 飯店）拿給使用者驗證設計方向
- 之後再補：飛機上、車站、拉麵、居酒屋、便利店／藥妝、觀光、緊急

### 技術選擇
- Vanilla JS（跟 carb-calc 一致 stack），單檔 app.js 約 600 行
- Furigana 用 `{漢字|讀音}` 自訂語法 → JS 解析成 `<ruby>...<rt>...</rt></ruby>`
  - 同一份字串可餵 TTS（strip 成純漢字交給日文 voice）
- TTS：Web Speech API，listen `voiceschanged`，預設 rate 0.9
- 資料：每場景一個 JSON（`data/scenarios/{id}.json`）+ `data/index.json` 列出順序
- localStorage key：`jpt_state_v1`
- Service Worker：static cache-first + data swr

### 設計決策
- Bottom nav 3 個 tab（對話 / 單字 / 測驗），設定放 header 齒輪 → modal
- 場景進入：grid → 子頁（含返回鍵）。對話跟單字共用同一個場景 grid 渲染函式
- 對話：店員=朱赤頭像、顧客=紺頭像。每句獨立播放鈕、整段播放會 highlight 當前句並 scroll
- 隱藏中文：CSS blur 而非 display none（保留版面）
- 單字：filter pill (全部 / 待加強 / 已會) + 翻面模式 (中→日)
- 測驗：weak vocab 加權三倍出題，known 一半機率出
- 答錯自動加入 weak、答對且原本 weak → 移除

### 內容統計（v0.1.0）
- 機場：3 段對話 24 句、30 個單字
- 餐廳：3 段對話 26 句、32 個單字
- 飯店：3 段對話 25 句、33 個單字
- **總計：9 段對話 75 句、95 個單字**

### 漢字 furigana 校對重點
- 觀光 かんこう、滞在 たいざい、搭乗券 とうじょうけん
- 自動販売機 じどうはんばいき、何名様 なんめいさま
- 805号室 はちまるごごうしつ（旅館房號慣讀）
- ご馳走様でした ごちそうさまでした、申し訳ございません もうしわけございません
